### Integration Issues (cosmetic only)

1. **NOTF-01 — Comment notification card title:** `handleIssueCommentWebhook` uses `commentData.Issue` (UUID) as issue name. Card reads "Nuevo comentario: `<uuid>`" instead of human-readable title. Known limitation — Plane's comment webhook payload lacks parent issue name. Fix requires extra API call.

2. **NOTF-02 — Digest post URL format:** `digest.go` line 197 uses `projects/{UUID}/work-items/` format instead of the `/browse/` format used everywhere else. URL is valid but inconsistent.

3. **CREA-02 — Dead export:** `GetWorkItem(projectID, workItemID)` in `plane/work_items.go` is exported and tested but never called in production. The unfurl path uses `GetWorkItemBySequence` instead.
