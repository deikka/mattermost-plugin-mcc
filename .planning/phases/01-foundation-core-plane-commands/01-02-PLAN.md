1. **server/plane/types.go** — All Plane API structs:
   ```go
   // WorkItem represents a Plane work item (formerly "issue")
   type WorkItem struct {
       ID          string   `json:"id"`
       Name        string   `json:"name"`
       Description string   `json:"description_html,omitempty"`
       State       string   `json:"state"`       // state UUID
       StateDetail *State   `json:"state_detail,omitempty"` // when expanded
       Priority    string   `json:"priority"`    // none, low, medium, high, urgent
       Assignees   []string `json:"assignees"`
       Labels      []string `json:"labels"`
       Project     string   `json:"project"`
       ProjectDetail *Project `json:"project_detail,omitempty"`
       CreatedAt   string   `json:"created_at"`
       UpdatedAt   string   `json:"updated_at"`
       SequenceID  int      `json:"sequence_id"`
   }

   type CreateWorkItemRequest struct {
       Name        string   `json:"name"`
       Description string   `json:"description_html,omitempty"`
       State       string   `json:"state,omitempty"`
       Priority    string   `json:"priority,omitempty"`
       Assignees   []string `json:"assignees,omitempty"`
       Labels      []string `json:"labels,omitempty"`
   }

   type Project struct {
       ID          string `json:"id"`
       Name        string `json:"name"`
       Identifier  string `json:"identifier"`
       Description string `json:"description"`
       Network     int    `json:"network"` // 0=secret, 2=public
       CreatedAt   string `json:"created_at"`
   }

   type State struct {
       ID       string `json:"id"`
       Name     string `json:"name"`
       Color    string `json:"color"`
       Group    string `json:"group"`    // backlog, unstarted, started, completed, cancelled
       Sequence float64 `json:"sequence"`
   }

   type Label struct {
       ID    string `json:"id"`
       Name  string `json:"name"`
       Color string `json:"color"`
   }

   type Member struct {
       ID          string      `json:"id"`
       Member      MemberUser  `json:"member"`
       Role        int         `json:"role"`
   }

   type MemberUser struct {
       ID          string `json:"id"`
       Email       string `json:"email"`
       DisplayName string `json:"display_name"`
       FirstName   string `json:"first_name"`
       LastName    string `json:"last_name"`
   }

   // PaginatedResponse wraps Plane API paginated results
   type PaginatedResponse struct {
       Results    json.RawMessage `json:"results"`
       TotalCount int             `json:"total_count"`
       NextCursor string          `json:"next_cursor"`
       PrevCursor string          `json:"prev_cursor"`
       NextPageResults bool       `json:"next_page_results"`
       PrevPageResults bool       `json:"prev_page_results"`
   }

   // APIError represents a Plane API error response
   type APIError struct {
       StatusCode int
       Message    string
       Detail     string
   }
   ```

2. **server/plane/client.go** — HTTP client:
   - `Client` struct: baseURL, apiKey, workspace, httpClient (10s timeout), cache *Cache
   - `NewClient(baseURL, apiKey, workspace string) *Client` — trims trailing slash from baseURL, creates http.Client with 10s timeout, initializes cache via NewCache()
   - `doRequest(method, path string, body interface{}) (*http.Response, error)` — constructs URL as `{baseURL}/api/v1/workspaces/{workspace}{path}`, sets `X-API-Key` and `Content-Type: application/json` headers. Marshals body if non-nil. Returns response.
   - `parseAPIError(resp *http.Response) *APIError` — reads body, returns structured error with status code
   - `IsConfigured() bool` — returns true if baseURL, apiKey, workspace are all non-empty
   - `UpdateConfig(baseURL, apiKey, workspace string)` — for config changes (called from OnConfigurationChange). Thread-safe with mutex.

3. **server/plane/projects.go** — Read operations:
   - `ListProjects() ([]Project, error)` — GET `/projects/`. Uses cache ("projects", 5 min TTL). Returns all projects in workspace.
   - `ListProjectStates(projectID string) ([]State, error)` — GET `/projects/{pid}/states/`. Cache key: "states_{pid}", 10 min TTL. Returns sorted by sequence.
   - `ListProjectLabels(projectID string) ([]Label, error)` — GET `/projects/{pid}/labels/`. Cache key: "labels_{pid}", 5 min TTL.
   - `ListProjectMembers(projectID string) ([]Member, error)` — GET `/projects/{pid}/members/`. Cache key: "members_{pid}", 5 min TTL.
   - `ListWorkspaceMembers() ([]Member, error)` — GET `/members/`. Cache key: "ws_members", 5 min TTL. Used by /task connect for email matching.

   Each method: check cache first, if miss call doRequest, handle pagination if needed (for Phase 1 with small teams, single page is likely sufficient — but handle `next_page_results` flag), decode, cache result, return.

4. **server/plane/work_items.go** — Work item operations:
   - `CreateWorkItem(projectID string, req *CreateWorkItemRequest) (*WorkItem, error)` — POST `/projects/{pid}/work-items/`. Returns created work item. NO caching.
   - `ListWorkItems(projectID, assigneeID string) ([]WorkItem, error)` — GET `/projects/{pid}/work-items/?assignees={id}&expand=state_detail,project_detail&per_page=10`. Returns list. NO caching (always fresh).
   - `GetWorkItemURL(projectID, workItemID string) string` — constructs browser URL: `{baseURL}/{workspace}/projects/{projectID}/work-items/{workItemID}`

5. **server/plane/cache.go** — Simple in-memory TTL cache:
   - `Cache` struct: `sync.RWMutex`, `items map[string]cacheItem`
   - `cacheItem` struct: `value interface{}`, `expiresAt time.Time`
   - `NewCache() *Cache`
   - `Get(key string) (interface{}, bool)` — returns value and true if exists and not expired. Deletes expired entries on access.
   - `Set(key string, value interface{}, ttl time.Duration)`
   - `Invalidate(key string)` — removes specific key
   - `InvalidateAll()` — clears entire cache (useful on config change)

Use explicit type assertions in callers (e.g., `cache.Get("projects")` returns `interface{}`, caller asserts `[]Project`).
