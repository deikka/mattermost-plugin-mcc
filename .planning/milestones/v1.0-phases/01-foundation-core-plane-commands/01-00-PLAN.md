  <files>server/testutil/helpers.go, server/testutil/mock_plane.go</files>
  <action>
Create the `server/testutil/` package (package name: `testutil`).

1. **server/testutil/helpers.go** — Common test setup utilities:
   - `SetupTestPlugin(t *testing.T) (*main.Plugin, *plugintest.API)`: Creates a Plugin instance with a mock API. Configures the mock API with reasonable defaults:
     - `API.On("LoadPluginConfiguration", mock.AnythingOfType("*main.configuration")).Return(nil).Run(func(args mock.Arguments) { ... })` — loads a test configuration with PlaneURL, PlaneAPIKey, PlaneWorkspace set to test values
     - `API.On("RegisterCommand", mock.Anything).Return(nil)`
     - Returns the plugin and mock API for further `.On()` calls in individual tests
   - `NewMockAPI(t *testing.T) *plugintest.API`: Creates a bare plugintest.API mock without default expectations
   - `TestConfig() *main.configuration`: Returns a test configuration struct with PlaneURL="http://localhost:8765", PlaneAPIKey="test-api-key", PlaneWorkspace="test-workspace"

   NOTE: Use `github.com/mattermost/mattermost/server/public/plugin/plugintest` for the mock API. The testutil package will need to import the main server package — if circular import issues arise, use an interface-based approach or keep helpers minimal.

   IMPORTANT: If the main package (`server/`) cannot be imported from a sub-package due to Go's `package main` restriction, restructure helpers to avoid importing `main`. Instead, provide factory functions that return mock APIs and config structs that callers wire up themselves. The Plugin struct instantiation stays in each test file since it's in `package main_test` or `package main`.

2. **server/testutil/mock_plane.go** — Mock Plane API HTTP server:
   - `MockPlaneServer` struct: wraps `httptest.Server`, stores configurable response maps
   - `NewMockPlaneServer(t *testing.T) *MockPlaneServer`: Creates an httptest.Server that handles:
     - `GET /api/v1/workspaces/test-workspace/projects/` — returns JSON array of 2 test projects
     - `GET /api/v1/workspaces/test-workspace/members/` — returns JSON array of 2 test workspace members (with emails)
     - `GET /api/v1/workspaces/test-workspace/projects/{pid}/states/` — returns JSON array of test states (backlog, started, completed)
     - `GET /api/v1/workspaces/test-workspace/projects/{pid}/labels/` — returns JSON array of test labels
     - `GET /api/v1/workspaces/test-workspace/projects/{pid}/members/` — returns JSON array of test project members
     - `GET /api/v1/workspaces/test-workspace/projects/{pid}/work-items/` — returns JSON array of test work items
     - `POST /api/v1/workspaces/test-workspace/projects/{pid}/work-items/` — returns created work item JSON (echoes name from request body)
     - Default: returns 404 for unmatched paths
   - `Close()` method: shuts down the test server
   - `URL() string`: returns the server base URL for use as PlaneURL in test config
   - `SetResponse(path string, statusCode int, body interface{})`: allows tests to override specific endpoint responses
   - Validates `X-API-Key` header on all requests — returns 401 if missing or wrong

   Use `net/http/httptest` and `github.com/gorilla/mux` (or plain http.ServeMux) for routing.
