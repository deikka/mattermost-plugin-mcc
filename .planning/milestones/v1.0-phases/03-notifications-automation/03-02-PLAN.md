**server/digest.go** -- Replace stubs with full implementation:

1. Add `digestJob *cluster.Job` field to Plugin struct in plugin.go (or use it from digest.go scope).

2. `startDigestScheduler() error`:
   ```go
   job, err := cluster.Schedule(
       p.API,
       "PlaneDigestScheduler",
       cluster.MakeWaitForRoundedInterval(1*time.Minute),
       p.runDigestCheck,
   )
   if err != nil {
       return fmt.Errorf("failed to schedule digest job: %w", err)
   }
   p.digestJob = job
   return nil
   ```

3. `stopDigestScheduler()`:
   ```go
   if p.digestJob != nil {
       if err := p.digestJob.Close(); err != nil {
           p.API.LogError("Failed to close digest job", "error", err.Error())
       }
   }
   ```

4. `runDigestCheck()`:
   - Use `p.API.KVList(0, 100)` to find all keys starting with `digest_config_`. For each key:
     - Parse `DigestConfig` from value
     - If `Frequency == "off"`, skip
     - Extract channelID from key (strip `digest_config_` prefix)
     - Get current server time: `now := time.Now()`
     - Check if digest is due:
       - Daily: `now.Hour() == config.Hour`
       - Weekly: `now.Hour() == config.Hour && now.Weekday() == time.Weekday(config.Weekday)`
     - Check `digest_last_{channelID}` to prevent re-posting:
       - Read last timestamp. If last post was within same hour (for daily) or same day (for weekly), skip.
       - Specifically: for daily, skip if `lastPostTime` is within the same calendar hour. For weekly, skip if within the same calendar day.
     - If due: build and post digest, then update `digest_last_{channelID}` with current Unix timestamp

   **Important:** Use a simple KVList approach. Since we expect a small number of digest configs (one per bound channel, typically <20 for a team of 2-10), iterating all keys is acceptable.

   **Alternative if KVList is not practical:** Maintain a "digest_registry" key that stores a JSON array of channelIDs with active digests. Update this array when SaveDigestConfig is called.

5. `buildDigestPost(channelID string, binding *store.ChannelProjectBinding) string`:
   - Fetch project work items: `p.planeClient.ListProjectWorkItems(binding.ProjectID)`
   - Group by state (reuse the pattern from `handlePlaneStatus`):
     - Open (backlog + unstarted), In Progress (started), Done (completed)
   - Calculate totals and progress percentage
   - Build markdown message:
     ```
     **Resumen del Proyecto: {projectName}** ({identifier})

     | Estado | Cantidad |
     |--------|----------|
     | :white_circle: Abierto | {open} |
     | :large_blue_circle: En Progreso | {inProgress} |
     | :white_check_mark: Hecho | {done} |

     **Progreso:** {progressBar} {percent}%
     **Total:** {total} tareas

     [Abrir en Plane]({projectURL})

     _Resumen automatico -- Configura con `/task plane digest off` para desactivar_
     ```
   - Return the markdown string

6. Post the digest as a VISIBLE channel post (not ephemeral):
   ```go
   post := &model.Post{
       UserId:    p.botUserID,
       ChannelId: channelID,
       Message:   digestContent,
   }
   if _, appErr := p.API.CreatePost(post); appErr != nil {
       p.API.LogError("Failed to post digest", "channel", channelID, "error", appErr.Error())
   }
   ```
